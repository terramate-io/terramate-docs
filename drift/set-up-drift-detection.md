# Set Up Drift Detection for Your Stacks

In this tutorial, you will configure and run drift synchronization so Cloud can track drift state.

## Prerequisites

- Terramate CLI authenticated with Terramate Cloud
- At least one managed stack

## Step 1: Run drift synchronization

Use your preferred workflow from the drift section (CLI, automation, or scripts) to send drift data.

## Step 2: Verify drift status

- Open Terramate Cloud Drift page
- Confirm stacks are marked with current drift state

## Step 3: Add notifications (optional)

- Connect Slack integration
- Enable drift notifications

## What you learned

- How drift data reaches Cloud
- Where to monitor and alert on drift
