# Glossary

## A

### Alert
An actionable incident in Terramate Cloud, usually created for failed deployments or detected drift.

## B

### Bundle
A reusable infrastructure capability package that teams can instantiate through self-service workflows.

## C

### Catalog
A local or remote source of bundles used in self-service and environment management workflows.

### Change Detection
Terramate capability that selects impacted stacks based on repository deltas and dependency filters.

### Collection
A logical grouping construct used in self-service workflows to organize related capabilities.

### Component
A reusable infrastructure building block used to standardize and compose environment definitions.

## D

### Deployment Synchronization
The process of sending deployment metadata from Terramate CLI to Terramate Cloud.

### Drift
A state where provisioned infrastructure no longer matches the declared IaC configuration.

## G

### Globals
Terramate configuration namespace for values shared across stack hierarchies.

## L

### Lets
Terramate configuration construct for local, computed values in configuration context.

## O

### Orchestration
Graph-aware command execution across stacks, supporting order, parallelism, and filtering.

## P

### Preview
A synchronized summary of proposed infrastructure changes, typically associated with pull requests.

## S

### Stack
The primary deployable unit in Terramate, representing an infrastructure scope where commands execute.

## T

### Terramate Cloud
The Terramate control plane that provides visibility, collaboration, alerts, and governance workflows.

### Terramate CLI
The open-source command-line tool for stack management, orchestration, and code generation.
